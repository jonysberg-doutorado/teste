Hard constraints Critérios eliminatórios veto automático se falhar
Os dados processados pelo LLM permanecem sob controle da organização ou em infraestrutura contratualmente autorizada? veto
Soberania de dados — LGPD art. 46, GDPR art. 32. API externa com dados de cliente = risco imediato.
O LLM ou seu provedor está sujeito a legislação de jurisdição incompatível com a do dado tratado? veto
Ex: modelo hospedado em servidor sujeito à CLOUD Act dos EUA com dados sensíveis de usuários brasileiros.
É possível gerar logs de uso auditáveis (quem usou, quando, com qual input)? veto
Auditabilidade é requisito do AI Act para sistemas de alto risco e da LGPD para tratamento de dados pessoais.
O provedor disponibiliza DPA (Data Processing Agreement) ou contrato equivalente? veto
Sem DPA, a organização não pode demonstrar base legal para o tratamento de dados via LLM externo.
Soft constraints Critérios ponderados por contexto avalie e documente
O time tem política interna documentada de uso aceitável de LLMs?
Análogo à Definition of Done — sem política, a responsabilidade recai individualmente sobre cada dev.
O time sabe quais tipos de dado NÃO podem ser enviados para um LLM externo?
Dados pessoais, segredos industriais, credenciais e código proprietário merecem categoria explícita de restrição.
Existe processo de revisão humana para outputs de LLM antes de chegar à produção?
Human-in-the-loop é requisito para sistemas de alto risco no AI Act e boa prática para qualquer uso em software crítico.
O provedor publica informações sobre como o modelo foi treinado e com quais dados?
Transparência sobre dados de treinamento é relevante para riscos de propriedade intelectual e viés.
Há mecanismo de opt-out para que os dados do cliente não sejam usados para treinar o modelo?
Vários provedores permitem isso via configuração de conta enterprise ou API — verifique antes de assinar.
Dimensões estratégicas Além do benchmark técnico documente a justificativa
O custo total de uso do LLM foi estimado além do preço por token? (infra, licença, integração, overhead de governança)
TCO de LLM frequentemente surpreende organizações que comparam apenas pricing público.
O time avaliou o risco de lock-in com o provedor escolhido?
Dependência de API proprietária sem portabilidade é risco estratégico de médio prazo.
A adoção do LLM foi comunicada formalmente à área jurídica e/ou ao DPO da organização?
A ausência de comunicação formal é, por si só, um risco operacional — não só regulatório.
Existe plano de revisão periódica da política de uso de LLM (mínimo semestral)?
O mercado de LLMs muda em meses. Uma política de 2023 pode estar desatualizada hoje.
O impacto da adoção do LLM na autonomia e desenvolvimento do time foi discutido explicitamente?
Sustentabilidade humana é dimensão de governança — não só produtividade imediata.
